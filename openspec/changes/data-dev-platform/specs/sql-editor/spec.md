## ADDED Requirements

### Requirement: 项目目录树

The system SHALL 为每个用户提供按"项目 → 目录 → 任务"层级组织的目录树，支持创建、重命名、移动、删除目录与任务节点。

#### Scenario: 用户在有权限的项目下新建任务

- **WHEN** 用户在某项目根目录或子目录右键选择"新建任务"，并填写任务名
- **THEN** 目录树立即出现该任务节点，任务状态为"草稿"，未发布到调度器

#### Scenario: 用户尝试操作无权限的项目

- **WHEN** 用户对未授权的项目节点发起任意写操作
- **THEN** 系统拒绝该操作并返回权限错误，目录结构保持不变

### Requirement: SQL 多 tab 编辑器

The system SHALL 提供多 tab 的 SQL 编辑能力，每个 tab 对应一个任务草稿，支持语法高亮、自动保存、撤销重做、未保存变更标记。

#### Scenario: 用户在 tab 中编辑 SQL

- **WHEN** 用户在编辑区输入或粘贴 SQL 内容
- **THEN** tab 标题显示未保存标记，并在用户停止输入后的设定时间内自动保存到草稿

#### Scenario: 用户关闭含未保存变更的 tab

- **WHEN** 用户点击关闭某个有未保存变更的 tab
- **THEN** 系统弹出确认提示，未确认前不丢弃变更

### Requirement: 变量面板

The system SHALL 在编辑器右侧提供变量面板，分类展示可用变量及其示例值，区分项目变量(发布时渲染)与运行时变量(driver 渲染)。

#### Scenario: 面板列出可用变量

- **WHEN** 用户打开任意 SQL tab
- **THEN** 变量面板显示当前项目的项目变量列表与平台承诺的运行时变量列表，每个变量给出名称、类型(项目/运行时)、示例值和说明

#### Scenario: 用户在 SQL 中使用未声明的变量

- **WHEN** 用户的 SQL 引用了既不在项目变量也不在运行时变量集合中的 `${...}` 占位符
- **THEN** 编辑器在该位置显示警告标记，指出变量未声明

### Requirement: 所见即所得预览

The system SHALL 提供"预览"操作，把当前 SQL 渲染为指定 `biz_date` 下的最终执行 SQL，渲染语义与发布后实际执行链路一致。

#### Scenario: 用户预览 SQL

- **WHEN** 用户点击"预览"，并在弹窗中选择一个 `biz_date`
- **THEN** 系统返回完整渲染后的 SQL 文本，其中项目变量与运行时变量均按发布+运行链路的同样规则替换

#### Scenario: 预览 SQL 含未定义变量

- **WHEN** 待预览 SQL 中存在未在任何变量集合中声明的占位符
- **THEN** 预览拒绝执行并提示具体未定义变量名，避免生成误导性的预览结果

### Requirement: 草稿与发布隔离

The system SHALL 区分"草稿"与"已发布"两种状态，仅"已发布"的内容会被调度器使用。

#### Scenario: 用户保存草稿

- **WHEN** 用户保存 SQL 编辑变更但未点击"发布"
- **THEN** 调度器仍按上一个已发布版本运行，草稿不影响生产任务实例
